## 🔴 RED Findings (Critical — Fix Immediately)

### R-002 — Cypher Injection via Incomplete Blocklist
**File:** [src/graph/cypher_healer.py](src/graph/cypher_healer.py#L73-L81)  
**Issue:** `_BLOCKED_KEYWORDS` misses `CREATE USER`, `ALTER USER`, `CALL dbms.security.*`  
**Impact:** LLM-generated Cypher could create rogue Neo4j admin user  
**Fix:** Switch to positive allowlist: only allow `MERGE`, `MATCH`, `RETURN`, `WITH`, `WHERE`, `SET`, `UNWIND`

### R-008 — SSRF via Unvalidated base_url
**File:** [src/config/model_builders.py](src/config/model_builders.py#L240-L260)  
**Issue:** `base_url` from user/env passed directly to `ChatOpenAI()` — no scheme/host validation  
**Impact:** SSRF to internal services, AWS metadata endpoint, attacker-controlled servers  
**Fix:** Validate URL scheme (`http`/`https` only), reject private IPs and metadata endpoints

### F-001 — SSRF via lmstudio_base_url in PipelineConfig
**File:** [src/api/models.py](src/api/models.py#L54)  
**Issue:** `lmstudio_base_url` field accepts arbitrary strings without validation  
**Impact:** Attacker redirects LLM calls to `file://`, `gopher://`, internal services  
**Fix:** Add Pydantic `@model_validator` restricting to `http`/`https` + private IP ranges only

### D-001 — Undefined `_logger` Variable in FastAPI App
**File:** [src/api/app.py](src/api/app.py#L93)  
**Issue:** `_logger` used but never imported → `NameError` if `CORS_ORIGINS="*"`  
**Fix:** Add `from src.config.logging import get_logger; _logger = get_logger(__name__)`
