## RED — Critical (Must Fix)

### R-01: `merge_results` does NOT implement Reciprocal Rank Fusion
- **Agent:** logic-auditor
- **File:** `src/retrieval/hybrid_retriever.py:441-468`
- **Issue:** Docstring claims RRF. Actual code: `max(score)` per node across sources. No rank-based scoring. BM25/vector scores compared directly (apples-to-oranges). Graph neighbours always lose (fixed 0.5).
- **Fix:** Replaced with true RRF: `score = sum(1/(k+rank_i) for each source)`, default `k=60`. Updated unit tests.
- **Impact:** Thesis validity — core retrieval contribution claim was incorrect.

### R-02: Auth bypass when `API_KEY` unset
- **Agent:** exploit-hunter
- **File:** `src/api/auth.py:56-81`
- **Issue:** When `API_KEY` env var empty/missing, ALL endpoints unprotected including destructive `DELETE /graph`.
- **Fix:** Added prominent box-drawing WARNING banner logged at first unauthenticated request. Auth disabled-by-default kept for dev flexibility but now impossible to miss in logs.

### R-03: Arbitrary env var injection via `POST /api/v1/config`
- **Agent:** exploit-hunter
- **File:** `src/api/app.py:169-202`
- **Issue:** Blocklist approach incomplete — `HOME`, `USER`, `SHELL`, `PYTHONHOME`, etc. not blocked.
- **Fix:** Replaced blocklist with allowlist derived from `Settings.model_fields`. Sensitive keys explicitly blocked even if in Settings. Uppercase env var aliases accepted.
