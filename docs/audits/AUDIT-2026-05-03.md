## 🔴 RED — Critical Findings (8)

### R-001 — Cypher Injection via String Concatenation
- **Agent:** exploit-hunter
- **File:** [src/graph/cypher_healer.py](src/graph/cypher_healer.py#L72)
- **Issue:** `f"EXPLAIN {cypher}"` enables arbitrary Neo4j query injection through LLM output
- **Impact:** DB corruption, schema destruction, data exfiltration
- **Likelihood:** 5/5
- **Fix:** Use Neo4j EXPLAIN via parameter binding or whitelist Cypher AST before execution

### R-002 — Unauthenticated DELETE Endpoint (Weak Protection)
- **Agent:** exploit-hunter
- **File:** [src/api/demo_router.py](src/api/demo_router.py#L510)
- **Issue:** DELETE `/graph` only requires `?confirm=true` query param. If `API_KEY` unset, endpoint is public
- **Impact:** Total data loss (MATCH (n) DETACH DELETE n)
- **Likelihood:** 4/5
- **Fix:** Require API_KEY for DELETE ops; implement 2-factor delete (request token + confirm)

### R-003 — Path Traversal in File Upload/doc_paths
- **Agent:** input-fuzzer
- **File:** [src/api/demo_router.py](src/api/demo_router.py#L42)
- **Issue:** `_to_abs()` path validation incomplete; symlinks can escape repo root
- **Impact:** Arbitrary file read outside repo directory
- **Likelihood:** 3/5
- **Fix:** Validate against whitelist of allowed directories; resolve symlinks before startswith check

### R-004 — Cypher Backtick Injection in Relationship Type
- **Agent:** input-fuzzer
- **File:** [src/graph/kg_registry.py](src/graph/kg_registry.py#L333)
- **Issue:** `f"MERGE (a)-[r:\`{rel_type}\`]->(b)"` with unsanitized rel_type from snapshot JSON
- **Impact:** Neo4j injection, unauthorized graph mutation
- **Fix:** Validate rel_type against whitelist; use parameterized approach

### B-001 — Missing State Preservation in Null Proposal Path
- **Agent:** logic-auditor
- **File:** [src/graph/validation_nodes.py](src/graph/validation_nodes.py#L36)
- **Issue:** When proposal is None, return dict loses `best_proposal` → Actor-Critic fallback broken
- **Impact:** Table mapping fails silently or uses wrong concept
- **Fix:** Add `"best_proposal": state.get("best_proposal")` to return

### B-002 — Blocked Keywords Detected but Cypher Still Executed
- **Agent:** logic-auditor
- **File:** [src/graph/build_nodes.py](src/graph/build_nodes.py#L230)
- **Issue:** After detecting blocked keywords (DROP/DELETE), `exec_cypher` still set to `llm_cypher` and executed
- **Impact:** Dangerous Cypher statements bypass safety guard → data corruption
- **Fix:** After blocked_keywords loop, if `cypher_failed=True`, use fallback `build_upsert_cypher()`

### I-002 — State Field Name Typo (Silent Data Loss)
- **Agent:** integration-inspector
- **File:** [src/generation/query_graph.py](src/generation/query_graph.py#L289)
- **Issue:** Node writes `"retrieved_filtered_by_threshold"` but schema expects `"retrieval_filtered_by_threshold"` → value never read
- **Impact:** Boolean always False in API response (silent data loss)
- **Fix:** Rename to consistent `"retrieval_filtered_by_threshold"` in lines 289 and 457

### Config-R-003 — Provider Base URLs Hardcoded
- **Agent:** config-drifter
- **File:** [src/config/provider_detection.py](src/config/provider_detection.py#L52)
- **Issue:** All LLM provider URLs hardcoded (OpenRouter, Groq, Together, etc.). Air-gapped/proxy deployments broken.
- **Impact:** Cannot deploy behind corporate proxy or use mirror endpoints
- **Fix:** Move to config.py with env var overrides per provider
