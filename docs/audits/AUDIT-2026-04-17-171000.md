## Findings

### YELLOW — Medium (Nice to Fix)

#### Y-001: Hardcoded localhost:8000 in nginx.conf — no env var override

- **Agent:** config-drifter
- **File:** `frontend/nginx.conf:15`
- **Issue:** `http://localhost:8000` hardcoded as upstream. `API_PORT` env var in `.env.example` is a dead variable — never read by serve_api.py or nginx.
- **Impact:** Port change requires editing two files (nginx.conf + serve_api.py --port arg) independently.
- **Suggested fix:** Make serve_api.py read `API_PORT` from env. Template nginx.conf with `envsubst` at start, or document the coupling.

#### Y-002: serve_api.py default port doesn't read API_PORT env var

- **Agent:** config-drifter
- **File:** `scripts/serve_api.py`
- **Issue:** Port hardcoded as `argparse default=8000`. `.env.example` defines `API_PORT=8000` but it's never consumed.
- **Impact:** Setting `API_PORT=9000` in `.env` has no effect — misleading configuration.
- **Suggested fix:** Change default to `int(os.environ.get("API_PORT", "8000"))`.

#### Y-003: Inconsistent Neo4j password placeholder across docs

- **Agent:** config-drifter
- **File:** `.env.example`, `RUNNING_SERVICES.md`
- **Issue:** `.env.example` uses `thesis_password`, `RUNNING_SERVICES.md` quick-start uses `test_password`. Two different placeholder values for the same purpose.
- **Impact:** Confusing for new users. Not a security issue (both are placeholders).
- **Suggested fix:** Standardize to one placeholder (e.g. `changeme`) across all docs.

### GREEN — Low (Optional)

- **Issue:** Pre-existing — not introduced by this changeset. No way to configure LLM request timeout.
