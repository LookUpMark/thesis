## Findings

### RED — Critical (Must Fix)

#### R-001: Cypher Injection via `rel_type` f-string in Snapshot Import
- **Agent:** exploit-hunter
- **File:** `src/graph/kg_registry.py:324-325`
- **Issue:** `rel_type` from deserialized JSON snapshot interpolated directly into Cypher MERGE via f-string: `f"... MERGE (a)-[r:{rel_type}]->(b) SET r += $props"`. No validation or allowlisting of the relationship type.
- **Adversarial scenario:** Malicious snapshot JSON with `rel_type` = `MAPPED_TO]->(b) DETACH DELETE b //` executes arbitrary destructive Cypher against Neo4j. Triggered via `POST /demo/kg/snapshots/{id}/load`.
- **Suggested fix:** Validate `rel_type` against allowlist of known relationship types (`MAPPED_TO`, `HAS_ATTRIBUTE`, `REFERENCES`, `MENTIONS`, `DESCRIBED_BY`, `PART_OF`, `INSTANCE_OF`, `CONTAINS_CHUNK`). Reject unknown types. Use backtick quoting as defense-in-depth.

#### R-002: HITL Routes to `"Generate_Cypher"` (PascalCase) — Node Does Not Exist
- **Agent:** integration-inspector
- **File:** `src/mapping/hitl.py:114,133,139,152`
- **Issue:** `Command(goto="Generate_Cypher")` but node registered as `"generate_cypher"` (snake_case) in `builder_graph.py:226`. LangGraph node names are case-sensitive.
- **Adversarial scenario:** Any HITL approval or correction crashes the pipeline with `ValueError: Graph has no node named "Generate_Cypher"`.
- **Suggested fix:** Change all 4 occurrences of `goto="Generate_Cypher"` to `goto="generate_cypher"`.

#### R-003: HITL Routes to `"End"` — Node Does Not Exist
- **Agent:** integration-inspector
- **File:** `src/mapping/hitl.py:118,156`
- **Issue:** `Command(update={"rejected": True}, goto="End")` but no node named `"End"` in the builder graph. The graph uses `END` (LangGraph constant) and `"save_trace"` as terminal routing.
- **Adversarial scenario:** HITL rejection crashes the pipeline with `ValueError`.
- **Suggested fix:** Change `goto="End"` to `goto="save_trace"` (which has an edge to `END`). Also add `"rejected": bool` to `BuilderState` TypedDict.

#### R-004: `_node_generate_cypher` Crashes on `KeyError` When `mapping_proposal` is `None`
- **Agent:** logic-auditor
- **File:** `src/graph/build_nodes.py:78`
- **Issue:** `proposal: MappingProposal = state["mapping_proposal"]` bare access crashes with `KeyError` if validation exhausted retries without producing a valid proposal. `_route_after_validate` still routes to `"generate_cypher"` when `mapping_proposal` is `None` because `current_table` is still set.
- **Adversarial scenario:** Actor-Critic exhausts `max_reflection_attempts` with only Pydantic-invalid proposals → pipeline crashes instead of gracefully terminating.
- **Suggested fix:** Add None guard at top of `_node_generate_cypher`. Fix `_route_after_validate` in `validation_nodes.py` to route to `"save_trace"` when `mapping_proposal` is `None` and `current_table` is not `None`.

### ORANGE — High (Should Fix)

#### O-001: Auth Silently Disabled When `API_KEY` Unset (Fail-Open)
- **Agent:** exploit-hunter
- **File:** `src/api/auth.py:60-63`
- **Issue:** No `API_KEY` env var → all endpoints unprotected including destructive ones: `DELETE /demo/graph`, `POST /api/v1/config` (env var override), `POST /demo/kg/snapshots/{id}/load` (Cypher injection).
- **Suggested fix:** Log WARNING at startup when auth disabled. Consider requiring explicit `AUTH_DISABLED=true` opt-out for non-local deployments.

#### O-002: File Read via Arbitrary Absolute Paths
- **Agent:** exploit-hunter
- **File:** `src/api/demo_router.py:141,179`; `src/api/ablation_router.py:88-91`
- **Issue:** `_to_abs()` passes absolute paths through without validation. Can read any file the process user has access to (e.g., `/etc/passwd`, SSH keys).
- **Suggested fix:** Restrict resolved paths to a configurable base directory (e.g., `DATA_DIR`). Reject paths with `..` components or outside allowed dirs.

#### O-003: Unpinned Dependencies — All Use `>=` Without Upper Bounds
- **Agent:** dependency-sniper
- **File:** `pyproject.toml:10-37`
- **Issue:** All dependencies use `>=X.Y` with no upper bound. Only `transformers` and `spacy` have upper bounds. Supply chain risk from breaking or compromised package updates.
- **Suggested fix:** Generate lock file with `pip-compile` or use `>=X.Y,<X+1.0` semver ranges for all dependencies.

#### O-004: `_node_heal_cypher` Crashes on `KeyError` When `mapping_proposal` is `None`
- **Agent:** logic-auditor
- **File:** `src/graph/build_nodes.py:100`
- **Issue:** Same bare `state["mapping_proposal"]` access as R-004 but in the Cypher healing node.
- **Suggested fix:** Add None guard consistent with R-004 fix.

#### O-005: `sqlite_checkpoint_path` Not Declared in `Settings`
- **Agent:** integration-inspector
- **File:** `src/graph/builder_graph.py:273`
- **Issue:** `settings.sqlite_checkpoint_path` accessed but not declared in `Settings` class. Will raise `AttributeError` when `production=True`.
- **Suggested fix:** Add `sqlite_checkpoint_path: str` field to `Settings` in `settings.py`.

#### O-006: Stale `_settings` in `hitl.py` After Runtime Reload
- **Agent:** integration-inspector
- **File:** `src/mapping/hitl.py:23`
- **Issue:** Module-level `_settings = get_settings()` evaluated once at import time. After `/api/v1/config` overrides env vars, HITL uses stale `confidence_threshold`.
- **Suggested fix:** Call `get_settings()` inside `should_interrupt()` instead of using module-level cache.

#### O-007: No Neo4j Driver Reuse — 16 Separate Instantiations
- **Agent:** performance-profiler
- **Files:** builder_graph.py (5), build_nodes.py (2), demo_router.py (3), query_graph.py (1), retrieval_nodes.py (1), expansion_nodes.py (1), kg_registry.py (2)
- **Issue:** Each `with Neo4jClient() as client:` creates new driver + `verify_connectivity()` call. Driver has internal pooling but instantiation overhead is significant per-request.
- **Suggested fix:** Implement singleton `Neo4jClient` with lazy initialization. Context manager reuses existing driver.

#### O-008: N+1 MENTIONS Repair Loop
- **Agent:** performance-profiler
- **File:** `src/graph/builder_graph.py:599-626`
- **Issue:** Individual `execute_cypher` per BusinessConcept per chunk index. 100 concepts x 50 chunks = 5,000 separate queries.
- **Suggested fix:** Batch with `UNWIND` Cypher statement collecting all (concept, chunk_index) pairs.

### YELLOW — Medium (Nice to Fix)

#### Y-001: Unrestricted Environment Variable Override via Config Endpoint
- **Agent:** exploit-hunter
- **File:** `src/api/app.py:168-191`
- **Issue:** `POST /api/v1/config` accepts arbitrary `dict[str, str]` and applies to `os.environ` with no allowlist. Could set `API_KEY=` (empty, disabling auth), `PATH=`, `PYTHONPATH=`, `LD_LIBRARY_PATH=`.
- **Suggested fix:** Add allowlist of configurable keys. Block sensitive keys.

#### Y-002: LLM-Generated Cypher Executed Directly (Prompt Injection Risk)
- **Agent:** exploit-hunter
- **File:** `src/graph/build_nodes.py:209-224`
- **Issue:** LLM-generated Cypher passes EXPLAIN syntax check but not semantic validation. A syntactically valid `DETACH DELETE` from prompt injection would execute.
- **Suggested fix:** Add keyword blocklist (`DETACH DELETE`, `DROP`, `REMOVE`, `DELETE` at top level) before execution.

#### Y-003: Hardcoded Default Neo4j Password `"neo4j"`
- **Agent:** config-drifter
- **File:** `src/config/settings.py:34`
- **Issue:** Default password `"neo4j"` is the most common Neo4j default. If env var not overridden, app connects with well-known credential.
- **Suggested fix:** Remove default, require `NEO4J_PASSWORD` env var or raise on empty.

#### Y-004: No Rate Limiting on Auth Attempts
- **Agent:** exploit-hunter
- **File:** `src/api/auth.py:73`
- **Issue:** `secrets.compare_digest` is used (good) but no rate limiting. Brute-force at line speed possible.
- **Suggested fix:** Add simple in-memory rate limiter (5 attempts/minute per IP).

#### Y-005: Verbose Exception Strings in HTTP Responses
- **Agent:** input-fuzzer
- **File:** `src/api/demo_router.py:387,525,685,728,830`
- **Issue:** Raw exception messages including Neo4j connection info, file paths, internal details leaked to clients.
- **Suggested fix:** Return generic error messages. Log details server-side only.

#### Y-006: Permissive CORS `allow_origins=["*"]`
- **Agent:** exploit-hunter
- **File:** `src/api/app.py:85-91`
- **Issue:** Any website can make API calls to the server from a user's browser.
- **Suggested fix:** Restrict to frontend origin or make configurable via env var.

#### Y-007: Grader/Critic Default to Pass on ALL Failures
- **Agent:** logic-auditor
- **File:** `src/generation/hallucination_grader.py:110-118`; `src/mapping/validator.py:115,123,133`
- **Issue:** On timeout or LLM failure, both grader and critic return `approved=True` / `grounded=True`. Hallucinated answers always accepted when grading LLM fails — opposite of safe default.
- **Suggested fix:** Log WARNING on fallback-to-pass. Consider configurable strict mode that returns `rejected` on failure.

#### Y-008: `"web_search"` Action from GraderDecision Silently Ignored
- **Agent:** logic-auditor
- **File:** `src/generation/routing.py:21`
- **Issue:** `GraderDecision` allows `action="web_search"` but routing only handles `"pass"` and `"regenerate"`. Falls through to `"finalise"`, silently skipping web search.
- **Suggested fix:** Remove `"web_search"` from `GraderDecision` Literal type or implement handling.

#### Y-009: SQLite Connections Leak on Graph Rebuilds
- **Agent:** logic-auditor
- **File:** `src/generation/query_graph.py:65`; `src/graph/builder_graph.py:273`
- **Issue:** `sqlite3.connect()` passed to `SqliteSaver` but never explicitly closed. On `reload_settings()` or graph rebuilds, old connections accumulate.
- **Suggested fix:** Track connection lifecycle, close on rebuild, or use connection pooling.

#### Y-010: `provenance_text[:300]` Crashes on `None`
- **Agent:** logic-auditor
- **File:** `src/mapping/rag_mapper.py:93`
- **Issue:** `Entity.provenance_text` typed as `str` but if constructed with `None`, slicing raises `TypeError`.
- **Suggested fix:** Use `(e.provenance_text or "")[:300]`.

#### Y-011: Unbounded Job Store Dict
- **Agent:** performance-profiler
- **File:** `src/api/jobs.py:13`
- **Issue:** `_store: dict[str, dict]` accumulates all jobs forever. Long-running server with many pipeline runs = unbounded memory growth.
- **Suggested fix:** Add TTL-based eviction or max size cap.

#### Y-012: Sync Disk I/O in Async Upload Handler
- **Agent:** performance-profiler
- **File:** `src/api/demo_router.py:89`
- **Issue:** `temp_file.write_bytes(content)` is synchronous disk I/O on the async event loop.
- **Suggested fix:** Use `aiofiles` or `asyncio.to_thread`.

#### Y-013: FK Edges Written Individually
- **Agent:** performance-profiler
- **File:** `src/graph/build_nodes.py:274-290`
- **Issue:** Each FK edge written with separate `execute_cypher` call in a loop.
- **Suggested fix:** Use `execute_batch` or batch with `UNWIND`.

#### Y-014: f-String Cypher Construction in Match Clauses
- **Agent:** exploit-hunter
- **File:** `src/graph/kg_registry.py:293-316`
- **Issue:** `_match_frag` constructs Cypher MATCH clauses using f-strings. `label` from snapshot JSON not strictly validated.
- **Suggested fix:** Validate labels against hardcoded set before interpolation.

#### Y-015: 30/63 Modules (48%) Have No Test File
- **Agent:** test-gap-analyst
- **Issue:** 48% of source modules lack dedicated tests. High-priority gaps: `ablation_router`, `app`, `auth`, `build_nodes`, `demo_router`, `generation_nodes`, `retrieval_nodes`, `validation_nodes`.
- **Suggested fix:** Add unit tests for critical untested modules, starting with API auth and graph build nodes.
